Javascript: Novice to Ninja

  • CHAPTER 10 - TESTING AND DEBUGGING

    • - Stack traces show the sequence of function or method calls that led to the error

    - Error Objects: the constructor function takes an argument that used as the error message

    - There are seven error objects used for errors:

  • - EvalError: Retained for backward compatibility. Errors using global eval()
  • - RangeError: When a number is outside an allowable range
  • - ReferenceError: When a reference is made to an object that doesn't exist
  • - SyntaxError: An error in code syntax
  • - TypeError: When wrong type is used
  • - URIError: When a problem with URI
  • - InternalError: When error occurs in JS engine

    • - These are only properties consistent across browsers and safe to use in Error objects are:

  • - name: name of error type
  • - message: error description
  • - stack: contains stack trace

    • - try, catch, finally exception handling

  • try{}: wrap code to be tested for an exception
  • catch(error): Code inside will only run if an exception occurs in try block
  • finall{}: This code will always run after try or catch block, regardless of exception
