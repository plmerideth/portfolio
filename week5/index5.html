  • CHAPTER 12 - OBJECT-ORIENTED PROGRAMMING IN JAVASCRIPT

    • - Three main components of OOP - encapsulation, polymorphism, and inheritance

    - Encapsulation: keeping all programming logic inside an object, only exposing methods, without needing to know how it's done

    - Polymorphism: Various objects can share a method, but also has the ability to override with a more specific implementation

    - Inheritance: Taking an object that already exists and inherit its properties and methods

    - Classes: Defines a blueprint for an object. We can add new properties and methods to improve it. In contrast, prototype-based languages use an actual object as the blueprint

    - Constructor Functions: An alternative way to object literals to create objects

    - Class Declarations: Does exactly the same thing as a constructor function, but looks more like class-based programming languages

    - The .constructor property holds the name of the constructor function or class that created the object

    - Static Methods: The static keyword is used to declare static methods that are only available to the class directly. Instances of the class will not have static methods

    - Prototype property returns an object. You can add new properties and methods to the prototype will be included in all new instances of the prototype

    - The prototype object is "live" so any properties or methods that are added to it are available to all instances of the prototype, regardless of when they were created (before or after the new properties or methods were added)

    - Class declarations are recommended over constructor functions to avoid accidentally redefining the prototype

    - JavaScript checks to see if an object has it's own properties or methods and will use those, or keep looking up the prototype chain until it finds the property or method

    - Public methods and properties are available to be queried and set directly. Private methods and properties are only accessible inside the class declaration. You must use getter() and setter() methods to read and set

    - Prototypes form a chain. Most objects start with {} prototype and its properties and methods. Each subsequent prototype in the chain inherits all the properties and methods of the prior prototypes

    - The extends keyword allows creation of a sub-class or child class of the parent and it inherts all it's parents properties and methods, but can add it's own custom properties and methods

    - Polymorphism allows objects with the same method to be implemented differently

    - Monkey-patching is the practice of adding methods to JavaScripts built-in objects, such as Number, String, and Array. This is discouraged however

  • CHAPTER 15 - MODERN JAVASCRIPT

    • - JQuery: most popular of all JavaScript libraries

    - JQuery Adv = support of older browsers, especially IE

    - $ symbol is used as alias for the global JQuery object containing all JQuery methods

    - Underscore and Lodash are popular libraries

    - ES6 now supports native JavaScript modules to keep parts of code in self-contained files

    - MVC is a design pattern used in server-side languages. It consists of separating applications into 3 parts that interact with each other: Models, Views, Controllers

    - Models: objects that implement the functionality of creating, reading, updating, and deleting pieces of information (CRUD)

    - Views: Provide a visual representation of the application. For example, HTML on a web page and forms

    - Controllers: Link modesl and views together by communicating between them. They respond to events, process the info, and update the model and view.

    - MV Frameworks: Many JavaScript frameworks don't have separate controller code because it's mixed in with Model and Views. These frameworks are lumped into a category called MV

    - MV Frameworks include: Aurelia, Angular, and Ember

    - View Libraries: some libraries have been developed that focus on just the view layer of MVC. The libraries work by using a virutal DOM in memory that compares the virtual DOM with the actual DOM and only updates the elements that have changed.

    - Popular View libraries are: React and Vue.js

    - Package Manager: With so many modules available for JavaScript, a tool was needed to manage them.

    - A package is just a directory with one or more files of code and a package.json which contains info about the code

    - NPM (Node Package Manager) was developed soon after Node.js to allow JavaScript code to be shared. It has grown into the largest repository of code libraries available

    - NPM repository is huge (500K different packages). Use search command to find a package

    - To deploy JavaScript, minimize code. This combines all JS files into one compressed file.

    - Popular minifiers: YUI Compressor, Google's Closure, and UglifyJS
