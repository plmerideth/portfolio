  • CHAPTER 6 - THE DOCUMENT OBJECT MODEL

    • - Node lists are array-like objects, but not actual arrays

    - Node lists don't have normal array methods, like splice, slice, and join

    - Node lists can be converted to normal arrays using Array.from(document.images) or with spread operator

    - Document.querySelector() returns only the FIRST element that matches the CSS selector

    - Document.querySelectorAll() returns a node list of all elements that match

    - querySelector() method can be used on any element, not just document. You first have to get a reference to the element of interest, and then you can query that element. For example, const ul = document.querySelector('ul#roster'); Then call const batman=ul.querySelector('li#bats');

    - Classes can be set in two ways. First, using setAttribute() method. Second, it can be set directly using the className property, but writing to className will overwrite other existing classes (if multiple classes exist). Use classList.add() to add a class

    - Once elements are created using document.createElement(), they must be linked using appendChild()

    - CSS properties that have a dash, like background-color, must use camelCase when being changed because property names cannot have dashes. background-color becomes backgroundColor

    - getComputedStyle() can be used to read the style properties, but it can't read styles set with external stylesheets

    CHAPTER 7 - EVENTS

    - Javascript uses non-blcoking approach for event handling

    - addeEventListener() method is called on a node object. It can also be called without a node and is then applied to the global object

    - Whenever an event handler is triggered, the callback function is called and automatically passed an event object as a parameter
