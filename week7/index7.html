    • - Promises represent the future result of an async operation. They help simplify the process

    - Promises can be chained so that one has to complete before the next function begins

    - The async keyword can be added in front of async functions so they act like synch functions.

    - Closures allow you to keep local variables in scope outside of the block they were created in by returning a function which references the variables to an external function

    - Functional programming is a programming paradigm, like object-oriented, or procedural. It uses 'pure functions' which make it easier to create more complex tasks and can be easily updated if needed

    - Currying is a process that involves the partial application of functions (Not all arguments supplied to the function call). It allows you to turn a single function into a series of functions

  • CHAPTER 13 - AJAX

    • - Same-Origin Policy: in browsers, it blocks all requests from a domain that is different from the page making the request.

    - CORS (Cross-origin Resource Sharing). It allows resources to be requested from another website outside the original domain. Works by using HTTP headers to indicate which domains can receive data.

    - Fetch API: A standard to request and send asynchronous data across a network.

    - Fetch API provides a global fetch() method that has one mandatory argument, which is the URL of the desired resource. fetch('https://example.com/data').then(//Code to handle the response).catch(//code to run if server returns an error);

    - The fetch() method returns a promise that resolves to the response returned from the URL

    - A response object is returned when the promise if fulfilled. These objects have a number of properties and methods.

    - the response.ok property will return true if HTTP status code property is between 200 and 299. The promise will only be rejected if a network error occurs, this does not include 404 not found errors.
